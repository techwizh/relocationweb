Executive summary

Relocate is a web platform for booking moving vehicles in Kenya.

Customers book online, pay via M-Pesa, chat with drivers, and track moves live.

Drivers register, get approved by admin, receive jobs, and share GPS.

Admins manage the landing page, approve drivers, assign bookings, and monitor fleet.

Live stack: Vercel (frontend) + Render (backend API) + Neon (PostgreSQL)

Website: Vercel URL  |  API: relocationweb-api.onrender.com

    Architecture

    Production uses a split deploy:

    
┌─────────────────────────────────────────────────────────────┐
│  VERCEL (Frontend)                                          │
│  • Next.js pages (landing, book, admin UI, etc.)            │
│  • Proxies /api/* → Render via API_URL rewrite              │
│  • No database connection                                   │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTPS (API calls + cookies)
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  RENDER (Backend)                                           │
│  • Next.js API routes (/api/*)                              │
│  • Prisma + PostgreSQL (Neon)                               │
│  • M-Pesa STK Push + callback                               │

└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼

│  NEON (PostgreSQL)                                          │

└─────────────────────────────────────────────────────────────┘

    User roles & flows

    Customer

  • Visit landing page → Book a move
  • Choose city, sub-county, ward, vehicle, date, contact details
  • Pay with M-Pesa (or skip if SKIP_MPESA_PAYMENT=true)
  • Chat with driver, track live GPS
  • Optional: register at /register → My account lists all bookings

    • Driver

  • Register at /driver/register (photos, license, vehicle)
  • Wait for admin approval
  • Log in at /login (Driver tab)
  • Driver portal: toggle availability, accept assigned jobs, update status, share GPS, chat

    • Admin

  • Log in at /admin/login
  • Dashboard — stats overview
  • Content — edit landing page text and upload hero/gallery images
  • Drivers — approve/reject applications, set internal contact phone
  • Bookings — assign paid bookings to approved drivers
  • Fleet — live map of driver GPS

    • Authentication & sessions

    All sessions use httpOnly cookies signed with ADMIN_SESSION_SECRET.

    CookieRoleMax age
    relocate_admin_sessionAdmin8 hours
    relocate_customer_sessionCustomer30 days
    relocate_driver_sessionDriver7 days
    relocate_booking_{id}Guest booking access90 days

    Split deploy rule: ADMIN_USERNAME, ADMIN_PASSWORD, and ADMIN_SESSION_SECRET must be identical on Render and Vercel so login cookies work across the proxy.

    Guest booking cookies are set when a booking is created, allowing chat/track/pay without an account.

    M-Pesa payments

  • Customer submits booking → status PENDING_PAYMENT
  • /book/[id]/pay triggers STK Push via Daraja API
  • Customer enters PIN on phone
  • Safaricom calls MPESA_CALLBACK_URL on Render
  • Booking status → PAID; customer redirected to chat

    Set SKIP_MPESA_PAYMENT=true to bypass payment during testing (booking goes straight to PAID).

    Production (live money)

  • Complete Safaricom Go Live on Daraja portal
  • Set MPESA_ENV=production on Render
  • Use live shortcode, passkey, consumer key/secret
  • Set MPESA_CALLBACK_URL=https://relocationweb-api.onrender.com/api/mpesa/callback
  • Set SKIP_MPESA_PAYMENT=false

    Deployment

    See also: SPLIT-DEPLOY.md

    Render (backend)

    
Build:  npm install && npx prisma generate && npx prisma migrate deploy && npm run build
Start:  npm start
Health: /api/health

    Vercel (frontend)

    
Build:  prisma generate && next build
Env:    API_URL + admin secrets (no DATABASE_URL)

    Vercel next.config.ts rewrites /api/:path* → ${API_URL}/api/:path*.

    Custom domain

  • Add domain in Vercel → Settings → Domains (website)
  • Keep M-Pesa callback on Render URL (or proxy if using same domain)

    Troubleshooting

    ProblemLikely causeFix
    GitHub shows README onlyGitHub is code storageUse Vercel URL for the website
    /api/health fails on RenderBuild failed or service sleepingCheck Render logs; wait 30–60s on free tier
    Admin login fails on VercelSecrets don't match RenderSync ADMIN_PASSWORD and ADMIN_SESSION_SECRET
